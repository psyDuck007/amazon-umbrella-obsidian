# 优惠券与折扣规则

## 适用场景

用于规范 `优惠券与折扣规则` 相关的执行规则。

## 输入信息

| 输入 | 来源 | 是否必需 | 备注 |
|---|---|---|---|
| ASIN / SKU | Seller Central / 领星 | 是 | 确认优惠作用对象 |
| Coupon 金额或比例 | Seller Central | 是 | 计算最坏利润 |
| 是否允许与其他促销叠加 | Coupon 设置 | 是 | 只能降低部分叠加风险，不能替代前台验证 |
| 同期 Deal / 秒杀 / Price Discount / Prime Exclusive Discount | Seller Central | 是 | 需要检查是否可能叠加 |
| 利润底线 | 利润表 | 是 | 大件商品优先防止亏损 |

## 已确认规则

- 优惠券可能与其他价格工具在前台形成叠加展示，不能默认认为不会叠加。
- 如果 Coupon 设置了“不与其他优惠活动同时使用”，可降低与部分促销工具叠加的风险，但仍需以前台商品页、购物车和结算页最终价格为准。
- 大件罗马伞的 Coupon 不应只按点击率考虑，必须按最坏叠加情况计算利润。
- 活动开始前要检查同 ASIN 是否已有 active coupon、deal、价格折扣或其他促销。

## 条件规则

| 条件 | 动作 | 例外 |
|---|---|---|
| 秒杀或 Deal 价格已经较低 | 暂停 Coupon 或降为小额 Coupon | 若确认不叠加且利润充足，可保留 |
| Coupon 主要用于前台绿色券提升点击 | 优先使用小额券，例如 `$10-$20` | 需结合售价和利润底线 |
| 同 ASIN 同时存在多个优惠工具 | 按最坏情况叠加计算利润 | 前台确认不叠加后再按实际价评估 |
| Coupon 设置不叠加 | 仍需检查商品页、购物车、结算页 | 不同优惠工具可能不完全受该开关控制 |

## 禁止事项

- 禁止在未计算利润底线时叠加大额 Coupon 和秒杀/Deal。
- 禁止只看后台设置，不做前台最终价验证。
- 禁止把“设置了不叠加”当成所有优惠工具都绝对不叠加的事实。

## 输出结果

- 活动前输出最低可能成交价。
- 输出 Coupon 成本和 redemption fee 对利润的影响。
- 输出是否暂停、降低或保留 Coupon 的建议。

## 复盘指标

| 指标 | 观察周期 | 判断方式 |
|---|---|---|
| CTR | 活动期间 | 判断绿色券或活动标签是否提升点击 |
| CVR | 活动期间及后 3 天 | 判断折扣是否真正提升转化 |
| 单件毛利 | 活动期间 | 防止折扣叠加打穿利润 |
| 广告 ACOS / TACOS | 活动期间及后 7 天 | 判断促销是否带动自然单和广告效率 |

## 待验证认知

- Coupon 的“不与其他优惠活动同时使用”对 Lightning Deal、Best Deal、Prime Exclusive Discount、Price Discount 等不同工具的实际限制边界，需要以前台实测和 Seller Central 规则继续验证。
