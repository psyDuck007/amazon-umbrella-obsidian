# 秒杀-Z划算-会员日规则

## 适用场景

用于规范 `秒杀-Z划算-会员日规则` 相关的执行规则。

## 输入信息

| 输入 | 来源 | 是否必需 | 备注 |
|---|---|---|---|
| 活动类型 | Seller Central | 是 | Lightning Deal / Best Deal / 7-Day Deal / Prime 相关活动 |
| 活动价 | Seller Central | 是 | 需和当前售价、Coupon、利润底线合并计算 |
| 同 ASIN 优惠券 | Seller Central | 是 | 检查是否 active |
| 前台最终价 | 商品页 / 购物车 / 结算页 | 是 | 以买家视角验证 |

## 已确认规则

- 秒杀和 Coupon 有可能形成前台叠加或造成折扣过深，不能默认安全。
- 活动前必须检查同 ASIN 是否存在 active Coupon 或其他折扣工具。
- 如果秒杀价已经接近利润底线，优先暂停或降低 Coupon。
- 秒杀最后 `1` 小时可以配合广告冲刺，但必须先确认叠加优惠后的利润底线。

## 条件规则

| 条件 | 动作 | 例外 |
|---|---|---|
| 秒杀价后再叠加 Coupon 会低于利润底线 | 暂停 Coupon 或取消活动 | 若活动目标是清库存，需要单独决策记录 |
| 仅想用 Coupon 标签提升点击 | 使用小额 Coupon，并以前台结算价验证 | 不适合大额券 |
| 前台显示价格与后台预期不一致 | 暂停新增预算和促销动作，先核价 | 需要截图留档 |
| 秒杀最后 1 小时且利润仍可接受 | 配合广告加预算或加竞价 | 广告 ACOS 失控或库存不足时不加 |

## 禁止事项

- 禁止同时开启大额 Coupon 和低价秒杀而不做最坏价格计算。
- 禁止只按后台设置判断是否叠加。

## 输出结果

- 活动前最低成交价核算。
- Coupon 是否暂停、降低或保留。
- 活动期间价格检查记录。
- 最后一小时广告冲刺记录。

## 复盘指标

| 指标 | 观察周期 | 判断方式 |
|---|---|---|
| 秒杀期间订单数 | 活动期间 | 判断活动放量效果 |
| 单件毛利 | 活动期间 | 判断是否打穿利润 |
| 广告花费与自然单变化 | 活动期间及后 7 天 | 判断活动是否带动自然排名和后续销量 |
| 最后一小时广告冲刺效果 | 秒杀最后 1 小时及归因延迟后 | 判断加广告是否值得复用 |
| 退货/售后变化 | 活动后 14 天 | 判断低价放量是否带来误购或质量投诉 |

## 待验证认知

- 不同 Deal 类型与 Coupon stacking 设置的实际叠加边界，需要以前台测试、订单最终成交价和 Seller Central 当前规则确认。
