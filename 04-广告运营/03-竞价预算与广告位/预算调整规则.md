# 预算调整规则

## 适用场景

用于规范 `预算调整规则` 相关的广告规则。

## 输入信息

| 输入 | 来源 | 是否必需 | 备注 |
|---|---|---|---|
| 广告组预算消耗 | Amazon 广告后台 / 领星 | 是 | 用于判断市场流量和竞价有效性 |
| 店铺整体出单量 | Amazon 后台 / 领星 | 是 | 19:00 判断是否触发归因延迟保护补预算 |
| 点击次数 | Amazon 广告后台 / 领星 | 是 | 20:55 判断竞价合格条件之一 |
| 曝光量 | Amazon 广告后台 / 领星 | 是 | 20:55 判断竞价合格条件之一 |
| 广告出单数 | Amazon 广告后台 / 领星 | 是 | 凌晨自动加预算规则条件 |
| 当前竞价 | Amazon 广告后台 / 领星 | 是 | 判断是否需要加价或降价 |
| 秒杀/Deal 剩余时间 | Seller Central | 促销期必需 | 判断是否进入最后 1 小时广告冲刺 |
| 过去 7 天出单时间段 | 订单报表 / 领星 / 广告报表 | 每周复盘必需 | 用于调整晚上加预算时间和递增预算工具 |

## 已确认规则

- 每天中国时间 `15:00` 将目标广告组预算更新为 `$6`，用于观察预算消耗速度，推测当天罗马伞亚马逊市场整体流量情况。
- 中国时间 `19:00` 如果店铺整体出单量 `>= 2`，因为广告归因存在延迟，可能来自当前广告组出单，需要对已经有预算消耗的广告组增加 `$10-$15` 预算。
- 中国时间 `20:55` 检查每个广告组：预算耗尽、点击次数 `>= 2`、曝光量 `>= 300`，三项中满足任意一项即代表竞价基本合格。
- 如果中国时间 `20:55` 三项均未满足，该广告组需要增加竞价。
- 中国时间 `21:00` 将广告组预算恢复到正常预算。对应美西时间需区分夏令时和冬令时：夏令时约为 PDT `06:00`，冬令时约为 PST `05:00`。
- 中国时间 `22:00` 再次检测预算消耗：低于 `$18` 或超过 `$35`，都代表竞价或消耗节奏异常，需要复查并调整。
- 已在领星设置凌晨自动加预算规则，详见 [[09-规则案例与实验/06-已验证结论/广告-日内预算消耗判断市场流量与竞价有效性]]。
- 秒杀最后 `1` 小时可以对表现良好且承接匹配的广告组加广告，详见 [[09-规则案例与实验/06-已验证结论/广告-秒杀最后一小时加广告]]。
- 每周需要分析过去 `7` 天出单时间段和每个时间段的出单百分比，用于调整晚上加预算时间；部分表现差但仍有承接价值的广告组，可用递增预算工具在高出单时间段增加预算。

## 条件规则

| 条件 | 动作 | 例外 |
|---|---|---|
| 中国时间 15:00 | 将目标广告组预算更新为 `$6` | 不适用于当天暂停或不观察的广告组 |
| 中国时间 19:00，店铺整体出单量 `>= 2`，且广告组已有预算消耗 | 给该广告组预算增加 `$10-$15` | 没有预算消耗的广告组不按此规则补预算 |
| 中国时间 20:55，预算耗尽 / 点击 `>= 2` / 曝光 `>= 300` 任意满足 | 竞价暂定合格 | 仍需结合 ACOS 和转化质量复盘 |
| 中国时间 20:55，三项均未满足 | 增加广告组竞价 | 若广告组当天不应投放或关键词不相关，则先检查结构 |
| 中国时间 21:00 | 恢复正常预算 | 正常预算金额按当前广告组策略执行 |
| 中国时间 22:00，预算消耗 `< $18` | 通常判断为拿量偏低，检查是否需要增加竞价 | 如果当天市场整体流量弱，需要谨慎加价 |
| 中国时间 22:00，预算消耗 `> $35` | 通常判断为消耗偏高，检查是否需要降低竞价或控预算 | 如果 ACOS 和订单表现优秀，可继续观察 |
| 中国时间 01:00-04:00，预算耗尽且广告出单 `>= 1` | 领星自动给广告组预算增加 `$30`，每 `15` 分钟检测一次 | 如果 ACOS 失控，需调整自动规则 |
| 中国时间 05:00-07:00，预算耗尽且广告出单 `>= 2` | 领星自动给广告组预算增加 `$40`，每 `15` 分钟检测一次 | 如果 ACOS 失控，需调整自动规则 |
| 秒杀最后 1 小时，广告组承接秒杀 ASIN 且表现良好 | 补预算或适度提高竞价 | 若 ACOS 失控、库存不足或利润不足，不加 |
| 每周复盘发现过去 `7` 天订单集中在某些时间段 | 调整晚上加预算时间 | 需结合 ACOS、CVR、库存和利润 |
| 表现差广告组错过高出单时间段，但搜索词和变体承接仍合理 | 用递增预算工具在高出单时间段增加预算 | 若低相关、承接错误或 ACOS 失控，不加 |

## 禁止事项

- 不要只看预算耗尽就无条件加预算，必须结合订单数、ACOS、点击质量和广告位。
- 不要把 19:00 店铺整体出单直接等同于某个广告组已归因出单；该动作是归因延迟保护，需要后续复盘验证。
- 不要忽略美西夏令时/冬令时差异。
- 不要把单日预算消耗速度直接等同于市场真实需求。
- 不要在秒杀最后一小时给低相关词、非秒杀变体或承接错误广告组盲目加预算。
- 不要只因为某时间段订单多就无条件加预算；必须结合广告组质量、ACOS、CVR、库存和利润。
- 不要用递增预算工具补救低相关或承接错误的广告组。

## 输出结果

- 每日预算消耗观察结论。
- 19:00 因店铺整体出单触发的补预算广告组清单。
- 20:55 竞价合格/需加价广告组清单。
- 22:00 预算消耗异常广告组清单。
- 凌晨领星自动加预算触发记录。
- 秒杀最后一小时广告冲刺记录。
- 过去 `7` 天出单时间段占比。
- 晚上加预算时间调整建议。
- 递增预算工具候选广告组和后续复盘记录。

## 复盘指标

| 指标 | 观察周期 | 判断方式 |
|---|---|---|
| 预算消耗速度 | 每日 15:00-20:55 | 判断当天市场早段流量 |
| 店铺整体出单量 | 每日 19:00 | `>= 2` 触发归因延迟保护补预算 |
| 广告组预算消耗 | 每日 19:00 | 有消耗才补 `$10-$15` |
| 点击次数 | 每日 20:55 | `>= 2` 为竞价合格信号之一 |
| 曝光量 | 每日 20:55 | `>= 300` 为竞价合格信号之一 |
| 预算是否耗尽 | 每日 20:55 | 耗尽为竞价合格信号之一 |
| 预算消耗金额 | 每日 22:00 | `< $18` 或 `> $35` 触发复查 |
| 广告订单数 | 凌晨自动规则 | 判断是否允许自动加预算 |
| ACOS / CPC / CVR | 每日 | 判断加预算和调竞价是否有效 |
| 秒杀最后一小时花费和订单 | 促销期 | 判断冲刺加广告是否有效 |
| 出单时间段占比 | 每周 / 最近 7 天 | 判断晚上加预算时间和递增预算工具设置 |
| 递增预算工具触发后订单和 ACOS | 后续 7 天 | 判断高出单时间段补预算是否有效 |

## 待验证认知

- 15:00-20:55 的预算消耗速度与当天罗马伞整体市场流量的量化对应关系。
- 19:00 触发补预算后，最终广告归因订单与实际 ACOS 的对应关系。
- 22:00 异常消耗后的具体竞价调整幅度。
- 秒杀最后一小时加广告的具体预算和竞价幅度。
- 过去 `7` 天出单时间段统计口径。
- 递增预算工具的具体预算增幅和触发时间。
