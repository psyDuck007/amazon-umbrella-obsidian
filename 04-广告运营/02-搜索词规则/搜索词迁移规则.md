# 搜索词迁移规则

## 适用场景

用于规范 `搜索词迁移规则` 相关的广告规则。

## 输入信息

| 输入 | 来源 | 是否必需 | 备注 |
|---|---|---|---|
| 搜索词报告 | Amazon 广告后台 / 领星 | 是 | 用于发现可迁移的新词 |
| 最近 7 天表现 | 广告报表 | 是 | 判断是否值得迁移 |
| 关键词相关性 | 人工判断 / 关键词库 | 是 | 判断是否和产品、尺寸、底座、灯光、场景匹配 |

## 已确认规则

- 每周需要从搜索词报告中拓展新词，特别是精准长尾词组。
- 有点击、有出单或高度相关的长尾搜索词，可以迁移到更清晰的精准或词组投放结构。
- 长尾词迁移时必须确认对应广告变体可以承接该词的尺寸、底座、灯光和场景需求。
- 自动广告组中出单数据比较好的词，可以拿到手动组来打。
- 用户搜索词数据除了用于否词，也用于判断是否需要单独拿出来打精准广告组。
- 同一个手动广告组里的词，流量体量尽量相当。

## 条件规则

| 条件 | 动作 | 例外 |
|---|---|---|
| 搜索词高相关且有订单 | 优先迁移为精准匹配 | 如果流量极小，可先放入词组匹配观察 |
| 搜索词高相关、有点击但无单 | 可加入候选长尾词，低预算测试 | 如果点击已很多且转化差，先诊断价格/Listing/变体 |
| 自动广告组搜索词出单表现好 | 迁移到手动组继续测试 | 需避免和现有手动组重复、内耗 |
| 用户搜索词具备独立测试价值 | 可单独拆精准广告组 | 需要确认预算足够，且词的流量体量适合独立测试 |
| 搜索词包含明确尺寸、底座、灯光或场景 | 迁移到能承接的对应广告组或变体 | 如果无法确认承接变体，先放入待确认 |
| 搜索词相关性弱 | 不迁移，必要时否定 | 无 |

## 禁止事项

- 不要把低相关大词当作拓词方向。
- 不要把带底座词迁移到不能承接带底座需求的变体。
- 不要只因为搜索量大就迁移，必须先判断相关性和承接能力。
- 不要把流量体量差异过大的词随意放在同一个手动组，避免大词吃预算、小词测不出来。

## 输出结果

- 每周新增精准长尾词组清单。
- 迁移到精准匹配的搜索词清单。
- 迁移到词组匹配的搜索词清单。
- 自动广告组出单好、迁移到手动组的搜索词清单。
- 需要单独拆精准广告组的用户搜索词清单。
- 因承接不明确而暂缓迁移的搜索词清单。

## 复盘指标

| 指标 | 观察周期 | 判断方式 |
|---|---|---|
| 新词 Clicks | 最近 7 天 / 测试期 | 判断是否有流量 |
| 新词 Orders | 最近 7 天 / 测试期 | 判断是否能转化 |
| 新词 ACOS | 最近 7 天 / 测试期 | 判断投放效率 |
| 新词相关性 | 每周 | 判断是否继续保留 |

## 待验证认知

- 精准长尾词组进入精准匹配和词组匹配的优先级。
