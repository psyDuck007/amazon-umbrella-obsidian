# 促销期数据判断

## 适用场景

用于规范 `促销期数据判断` 相关的广告规则。

## 输入信息

| 输入 | 来源 | 是否必需 | 备注 |
|---|---|---|---|
| 活动 ASIN / SKU | Seller Central / 领星 | 是 | 确认广告承接对象 |
| 活动类型与结束时间 | Seller Central | 是 | 判断是否进入最后 1 小时 |
| 活动价和 Coupon 叠加 | Seller Central / 前台 | 是 | 防止广告加码后利润被打穿 |
| 广告组花费、订单、ACOS | 广告后台 / 领星 | 是 | 判断是否值得最后一小时加广告 |
| 当前库存 | Seller Central / 领星 | 是 | 避免活动末段推爆断货或不可售 |

## 已确认规则

- 秒杀最后 `1` 小时可以加广告，用于抓住促销结束前的购买紧迫感和活动流量。
- 秒杀倒计时会强化买家的时间压力，最后一小时更容易触发“抓住优惠”的下单行为。
- 最后一小时加广告应优先给秒杀 ASIN 对应的已出单广告组、核心成交词和高相关精准长尾词。
- 加广告前必须确认活动价、Coupon 叠加和广告成本后仍符合利润底线。

## 条件规则

| 条件 | 动作 | 例外 |
|---|---|---|
| 秒杀进入最后 1 小时，广告组已出单或表现稳定 | 补预算或适度加竞价 | ACOS 已明显失控则不加 |
| 秒杀进入最后 1 小时，核心词流量不足但相关度高 | 小幅加竞价或给预算 | 如果库存不足或利润不足，先控量 |
| 广告组承接的不是秒杀 ASIN 或变体 | 不按此规则加广告 | 除非有明确跨变体承接策略 |
| 最后一小时 CPC 飙升且无订单 | 停止继续加码，活动后复盘 | 不因活动即将结束盲目追流量 |

## 禁止事项

- 禁止把“秒杀最后一小时加广告”理解为所有广告组无条件加预算。
- 禁止给非秒杀变体、承接不匹配变体或低相关大词盲目加广告。
- 禁止在未核算 Coupon / 秒杀叠加后利润时加大广告。

## 输出结果

- 秒杀最后一小时加广告对象清单。
- 补预算或加竞价幅度。
- 活动结束后的广告花费、订单、ACOS、毛利复盘。

## 复盘指标

| 指标 | 观察周期 | 判断方式 |
|---|---|---|
| 最后一小时广告花费 | 秒杀最后 1 小时 | 判断冲刺成本 |
| 最后一小时广告订单 | 秒杀最后 1 小时及归因延迟后 | 判断冲刺有效性 |
| 最后一小时订单占比 | 活动结束后 | 判断倒计时是否带来额外转化压力 |
| ACOS / CPC / CVR | 秒杀最后 1 小时及后 24 小时 | 判断加广告质量 |
| 秒杀总订单和单件毛利 | 活动结束后 | 判断活动整体是否值得复用 |

## 待验证认知

- 不同活动类型、不同尺寸和带底座/不带底座版本，最后一小时加广告的最佳预算和竞价幅度仍需继续复盘。
- 需要统计秒杀最后一小时订单占比，验证倒计时机制在不同 ASIN 上是否稳定有效。
