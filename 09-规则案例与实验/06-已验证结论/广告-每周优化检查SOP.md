# 广告-每周优化检查SOP

## 状态

- 结论状态：已验证
- 记录日期：2026-06-11
- 来源：用户运营经验
- 适用模块：广告运营、每周优化、否定关键词、投放词分析、广告位分析、广告组诊断

## 原始结论

每周广告需要做以下事情：

1. 否定相关度低的关键词，或者有大量点击但不出单的关键词。
2. 查看各个投放词最近 `7` 天的表现。
3. 查看各个广告组不同位置的表现，包括首页、其他搜索位置、商品页面。
4. 拓展新词，特别是精准长尾词组。
5. 对于最近 `7` 天表现差的广告组，需要考虑是否增加广告关键词、增加广告变体、调整广告，或降价。
6. 周二作为广告稳定时期的固定广告复盘优化日；推新品期间节奏需要加快，每 `2` 天至少复盘一次。
7. 每周分析过去 `7` 天出单时间段和每个时间段出单百分比，用于调整晚上加预算时间；表现差但仍有承接价值的广告组，可用递增预算工具在高出单时间段增加预算。

## 复盘频率规则

| 场景 | 复盘频率 | 复盘重点 |
|---|---:|---|
| 稳定投放期 | 每周二固定复盘 | 否词、投放词最近 `7` 天表现、广告位表现、精准长尾拓词、表现差广告组处理 |
| 推新品期 | 每 `2` 天至少复盘一次 | 快速判断曝光是否起来、点击是否有效、预算是否卡住、核心词/长尾词是否需要补充、低相关流量是否需要及时否定 |

执行边界：

- 稳定期不要每天因为短期波动频繁大改，周二集中复盘可以保持广告结构稳定。
- 推新品期不能等到完整 `7` 天后才处理，因为新品早期需要更快发现流量不足、词不准、预算消耗异常和转化承接问题。
- 新品每 `2` 天复盘不等于每次都必须大幅改价或改竞价；优先做小步调整，并记录动作与后续指标。

## 执行 SOP

### 0. 先看最近 3 天广告动作反馈

复盘开始时先看最近 `3` 天广告数据，并对照自己近期做过的广告动作。

| 检查对象 | 判断重点 | 动作 |
|---|---|---|
| 最近 `3` 天整体数据 | Spend、Orders、Sales、ACOS、CPC、CVR 是否有明显变化 | 判断近期广告动作是否可能产生正面影响 |
| 最近 `3` 天操作记忆 | 是否做过加/减竞价、加词、否词、广告位调整、变体投放调整 | 把动作与数据变化对应起来 |
| 后续 AI 分析 | 抓取 `7-14` 天广告动作记录和数据变化 | 让 AI 自动分析哪些动作带来正面影响 |

最近 `3` 天数据只用于建立近期判断，不直接作为否定高相关词、关闭商品投放或大幅重构广告组的唯一依据。

### 1. 否定低相关和高点击无单关键词

| 检查对象 | 判断重点 | 动作 |
|---|---|---|
| 搜索词 | 是否与罗马伞、尺寸、底座、灯光、使用场景相关 | 低相关则否定 |
| 高点击无转化词 | `7-14` 天点击次数多但订单数据差，或约 `30` 天数据仍较差 | 先判断相关性；高相关优先降竞价，长期差再否定 |
| 误导性词 | 可能引来错误预期，如底座、尺寸、功能不匹配 | 优先否定或拆分投放 |

### 2. 查看投放词最近 7 天表现

每个投放词至少看：

| 指标 | 用途 |
|---|---|
| Spend | 判断消耗压力 |
| Clicks | 判断是否有足够点击 |
| Orders | 判断是否出单 |
| Sales | 判断销售贡献 |
| ACOS | 判断广告效率 |
| CPC | 判断竞价成本 |
| CTR | 判断点击吸引力 |
| CVR | 判断转化承接 |

加/减竞价主要看最近 `7` 天数据；涉及否词、加词、增加或关闭商品广告投放时，至少看 `14` 天数据，并把 `30` 天数据纳入考虑。

### 3. 查看广告组不同位置表现

每个广告组需要拆看：

| 位置 | 判断用途 |
|---|---|
| 首页 / Top of Search | 判断核心词抢位是否有效 |
| 其他搜索位置 / Rest of Search | 判断中低位流量是否有性价比 |
| 商品页面 / Product Pages | 判断竞品页面或关联流量是否有效 |

广告位置百分比调整主要看最近 `3` 天和 `7` 天数据。

### 3.1 查看同一广告投放下不同变体表现

同一个广告投放下，需要拆看不同变体各自的广告表现。

| 检查对象 | 判断重点 | 动作 |
|---|---|---|
| 不同尺寸变体 | Spend、Orders、Sales、ACOS、CVR 是否明显不同 | 决定是否增加或减少某个尺寸的广告投放 |
| 带底座 / 不带底座变体 | 搜索词和变体承接是否匹配 | 避免需求错配导致广告浪费 |
| 不同颜色变体 | 点击后是否能转化 | 判断是否继续投放、降竞价或调整承接 |

增加或关闭商品广告投放这类较重动作，至少看 `14` 天数据，并结合 `30` 天数据辅助判断。

### 3.2 查看 AMC 人群百分比表现

AMC 人群百分比表现主要看 `7` 天和 `14` 天数据。

| 判断结果 | 动作 |
|---|---|
| 人群表现好 | 考虑增加百分比 |
| 人群表现差 | 考虑减少百分比 |
| 样本不足 | 继续观察，不急于调整 |

### 4. 拓展新词和精准长尾词组

每周优化不只做否词，也要主动拓展新词，尤其是更容易精准承接的长尾词组。

| 新词来源 | 判断重点 | 动作 |
|---|---|---|
| 搜索词报告中有点击或出单的长尾词 | 是否包含尺寸、功能、底座、灯光、形状或场景 | 提取为精准或词组投放 |
| 竞品标题/五点/评论中的高相关表达 | 是否和产品事实一致 | 加入候选词库，测试精准长尾 |
| Listing 核心词组合 | 尺寸 + 品类 + 功能 + 场景 | 组合成长尾词组 |
| 高相关但流量不足的投放词 | 是否可扩展同义词、场景词、尺寸词 | 拓展相近长尾词 |
| 自动广告组出单好的用户搜索词 | 是否具备独立承接价值 | 迁移到手动组或单独精准广告组 |

优先拓展方向：

- 尺寸 + 品类：如 `9x12 cantilever patio umbrella`。
- 功能 + 品类：如 `cantilever umbrella with lights`。
- 底座 + 品类：只用于能承接的带底座变体。
- 场景 + 品类：如 pool deck、backyard、outdoor dining 等高相关场景。

同一个手动广告组里的词，流量体量尽量相当，避免大词吃掉预算后小词没有测试机会。

### 5. 处理最近 7 天表现差的广告组

表现差的广告组不要只做一个动作，需要按原因拆解：

| 可能问题 | 判断方式 | 可选动作 |
|---|---|---|
| 词太少或流量不足 | 曝光低、点击少、预算消耗慢 | 增加广告关键词 |
| 变体承接弱 | 有点击但转化弱，可能颜色/尺寸/带底座版本不匹配 | 增加或更换广告变体 |
| 广告设置不合适 | CPC 高、广告位差、预算消耗异常 | 调整竞价、预算、广告位加价 |
| 预算错过高出单时间段 | 过去 `7` 天出单时间段集中，但广告组预算没有覆盖主出单时段 | 调整晚上加预算时间，或用递增预算工具在高出单时间段加预算 |
| 价格承接弱 | 点击正常但 CVR 低，竞品价格更有优势 | 考虑降价、Coupon 或促销 |

## 风险与边界

- 不要只因为关键词 7 天无单就机械否定，必须先看相关性、点击量和产品承接问题。
- 对低相关词可以更果断否定；对高相关但无单词，应先判断是否是价格、Listing、变体或广告位问题，优先降竞价而不是直接否定。
- 能出单的词很多，约 `30` 天数据仍表现较差，或 `7-14` 天点击次数多但订单数据差的词，可以否掉；后续如果广告组数据不好，可以再考虑把这个词重新放出来。
- 查看广告位表现时，要区分首页、其他搜索位置、商品页面，不要只看广告组总 ACOS。
- 调整广告位置百分比时，主要看最近 `3` 天和 `7` 天数据。
- 调整晚上加预算时间时，必须看过去 `7` 天出单时间段和各时间段出单百分比。
- AMC 人群百分比调整主要看 `7` 天和 `14` 天数据。
- 拓展新词时优先拓展精准长尾词组，不要把低相关大词直接扩进广告组。
- 同一个手动组里的词尽量保持流量体量相当。
- 带底座、尺寸、灯光相关长尾词必须由对应变体承接，避免版本不匹配。
- 表现差的广告组如果缺流量，优先考虑增加关键词或变体；如果有流量无转化，再考虑调价、调 Listing 或否词。
- 罗马伞是季节性产品，`30` 天以上数据不作为当前广告动作的主要依据。

## 输出结果

每周优化后应输出：

- 最近 `3` 天广告动作与数据变化对照。
- 否定关键词清单。
- 最近 7 天表现好的投放词清单。
- 最近 7 天表现差的投放词清单。
- 最近 `14` 天和 `30` 天否词、加词、增加/关闭商品广告投放判断。
- 新增精准长尾词组清单。
- 自动广告组出单好、值得迁移到手动组的搜索词清单。
- 值得单独拆精准广告组的用户搜索词清单。
- 各广告组广告位表现结论。
- AMC 人群百分比调整建议。
- 过去 `7` 天出单时间段和各时间段出单百分比。
- 需要调整晚上加预算时间的广告组或店铺级策略。
- 适合用递增预算工具在高出单时间段补预算的广告组清单。
- 需要增加关键词的广告组。
- 需要增加或更换广告变体的广告组。
- 需要调价、Coupon 或促销承接的广告组。

推新品期间每 `2` 天复盘后应额外输出：

- 新品广告组曝光、点击、花费和订单是否达到预期。
- 核心词、精准长尾词是否需要新增或拆分。
- 是否存在低相关搜索词需要及时否定。
- 是否存在预算消耗过慢、预算过早耗尽或竞价明显不合格。
- 下一轮 `2` 天观察动作。

## 复盘指标

| 指标 | 观察周期 | 判断方式 |
|---|---|---|
| 搜索词相关性 | 每周 | 是否和产品、尺寸、底座、灯光、场景匹配 |
| 点击无单词 | 最近 7 天 | 点击多但无广告订单 |
| 新增长尾词组 | 每周 | 是否高相关、可承接、可测试 |
| 最近广告动作反馈 | 最近 3 天 | 判断动作是否可能产生正面影响 |
| 投放词 ACOS | 最近 7 天 | 判断效率 |
| 广告位 ACOS | 最近 7 天 | 首页/其他/商品页面分别判断 |
| 广告位置百分比 | 最近 3 天 / 最近 7 天 | 判断是否需要调整位置百分比 |
| 出单时间段占比 | 最近 7 天 | 判断晚上加预算时间是否需要调整 |
| 递增预算工具效果 | 后续 7 天 | 判断高出单时间段加预算是否有效 |
| AMC 人群百分比 | 最近 7 天 / 最近 14 天 | 判断是否增加或减少百分比 |
| 广告组 CVR | 最近 7 天 | 判断承接能力 |
| 广告组 Spend | 最近 7 天 | 判断消耗压力 |
| 否词 / 加词 / 增减商品投放 | 最近 14 天 / 最近 30 天 | 判断较重动作是否有足够数据支持 |
| 新品广告曝光/点击/花费/订单 | 每 2 天 | 判断推新品广告是否及时拿到有效流量 |
| 新品低相关搜索词 | 每 2 天 | 及时否定，避免新品期预算浪费 |

## 待补数据

- “大量点击但不出单”的具体点击阈值。
- “最近 7 天表现差”的 ACOS、CVR、Spend 或订单阈值。
- 精准长尾词组进入精准匹配或词组匹配的优先级规则。
- 不同尺寸、带底座/不带底座广告组是否使用不同判断阈值。
- 推新品期间每 `2` 天复盘时，曝光、点击、花费、订单、ACOS、CVR 的最小动作阈值。
- 广告动作记录如何自动抓取，并与 `7-14` 天数据变化对齐。
- AMC 人群百分比增加或减少的具体阈值。
- 出单时间段统计应按小时还是按运营时间段聚合。
- 递增预算工具在不同广告组上的触发阈值和预算增幅。
