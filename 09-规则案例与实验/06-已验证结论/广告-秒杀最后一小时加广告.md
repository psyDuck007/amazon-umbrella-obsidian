# 广告-秒杀最后一小时加广告

## 结论状态

- 状态：已验证结论
- 来源：用户 2026-06-15 口述
- 适用模块：广告运营、促销期数据判断、秒杀/Deal 配合投放

## 用户原始结论

> 广告 已验证 在秒杀最后一小时加广告

补充机制：

> 秒杀最后一小时，因为有倒计时的存在，往往会让买家为了抓住优惠下单。

## 适用场景

- 产品正在参加秒杀、Lightning Deal、Best Deal、7-Day Deal 或类似限时促销。
- 秒杀已进入最后 `1` 小时。
- 活动前台价格、Coupon 叠加和利润底线已经检查过。
- 广告组本身与参加秒杀的 ASIN / 变体匹配，不能用不相关广告组抢流量。

## 操作规则

秒杀最后一小时可以主动加强广告，以利用限时结束前的购买紧迫感和促销流量。

该动作的核心依据是：秒杀倒计时会强化买家的时间压力和错过优惠的心理成本。最后一小时如果广告仍能把高相关买家引入详情页，买家更容易在“优惠即将结束”的提示下完成下单。

可执行动作包括：

1. 给表现较好的相关广告组补预算，避免最后一小时断流。
2. 对核心成交词、高相关长尾词或已出单广告组适度提高竞价。
3. 优先给与秒杀 ASIN 完全匹配的广告活动加广告，不把预算加到非活动变体或承接不匹配变体。
4. 如果已有广告组 ACOS 明显失控，不因“最后一小时”无条件继续加广告。

## 判断优先级

| 优先级 | 广告对象 | 动作 |
|---|---|---|
| P1 | 秒杀 ASIN 对应的已出单广告组 | 优先补预算或小幅加竞价 |
| P1 | 核心成交词、精准长尾词、AMC 已验证购买词 | 优先保护曝光和点击 |
| P1 | 已经看过或点击过促销商品、仍可能受倒计时影响的高相关流量 | 优先避免广告断流 |
| P2 | 有点击、有加购迹象但暂未归因出单的高相关词 | 可小额测试 |
| P3 | 广泛大词、低相关词、非秒杀变体广告组 | 不因秒杀最后一小时盲目加广告 |

## 风险边界

- 加广告前必须确认秒杀价、Coupon、其他折扣叠加后仍不低于利润底线。
- 秒杀最后一小时加广告是促销期冲刺动作，不等于全天放开预算。
- 不能把预算加到承接不匹配的尺寸、颜色、灯光或带底座/不带底座变体。
- 如果最后一小时 CPC 快速升高、ACOS 恶化或无订单，不应持续加码。

## 复盘指标

| 指标 | 观察周期 | 判断方式 |
|---|---|---|
| 最后一小时广告花费 | 秒杀最后 1 小时 | 判断加广告成本 |
| 最后一小时广告订单 | 秒杀最后 1 小时及归因延迟后 | 判断是否带来直接转化 |
| ACOS / CPC / CVR | 秒杀最后 1 小时及活动后 24 小时 | 判断加广告质量 |
| 秒杀总订单变化 | 活动结束后 | 对比未加广告场次或前几小时表现 |
| 最后一小时订单占比 | 活动结束后 | 判断倒计时是否带来冲刺转化 |
| 单件毛利 | 活动结束后 | 检查促销价和广告叠加后是否仍可接受 |
