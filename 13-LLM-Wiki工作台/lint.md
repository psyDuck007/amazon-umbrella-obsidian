# LLM-Wiki健康检查

## 使用场景

当用户说“检查知识库”“整理一下结构”“有没有矛盾”“复盘一下最近规则”时，按本页执行。

## 检查顺序

1. 读取 [[13-LLM-Wiki工作台/index]]。
2. 读取 [[13-LLM-Wiki工作台/log]] 最近 10 条。
3. 检查 `12-收集箱与附件` 是否有未处理输入。
4. 检查 `09-规则案例与实验` 是否有实验缺少结果。
5. 检查各业务模块总览是否链接到新增规则。
6. 检查产品事实、Listing、客服话术中是否混入未确认信息。

## 检查项

| 类型 | 检查问题 | 处理方式 |
|---|---|---|
| 矛盾结论 | 同一主题是否存在相反规则 | 标记冲突，保留来源和日期，不直接删除 |
| 过期结论 | 旧结论是否被新数据推翻 | 移入 `09-规则案例与实验/07-已废弃结论` 或补充适用条件 |
| 孤立页面 | 页面是否没有任何入口链接 | 更新 `index.md` 或对应模块总览 |
| 待验证假设 | 假设是否长期没有实验计划 | 加入 [[13-LLM-Wiki工作台/wiki/问题队列/待补证据清单]] |
| 待确认事实 | 是否有产品事实缺少证据 | 移入 [[12-收集箱与附件/03-待确认产品信息]] |
| 合规风险 | 是否出现删评、诱导好评、未证实防风/认证/质保 | 移入禁区或售后合规规则 |
| 命名混乱 | 文件名是否无法看出主题和状态 | 按使用说明重命名或补充索引 |
| 缺少复盘指标 | 实验是否没有观察周期和指标 | 补充 Sessions、转化率、订单量、ACOS、退款率等 |

-

-
