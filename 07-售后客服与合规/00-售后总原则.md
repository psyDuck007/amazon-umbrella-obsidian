# 售后总原则

## 模块定位

本模块用于处理 JZAC / QZYD Amazon US 罗马伞售后、客服沟通、退款退货、补发、评价与申诉相关内容。

客服处理顺序必须是：

1. 先判断问题和证据。
2. 再选择处理动作。
3. 再撰写合规邮件。
4. 最后沉淀案例和待确认信息。

## 处理优先级

1. 先确认订单、ASIN、是否分箱、物流状态。
2. 判断问题类型：未收到、只收到一箱、缺件、破损、功能故障、风损、误解。
3. 根据规则选择解释、补发、退款、退货或申诉。
4. 客服回复必须合规、克制、可证明。

## 客户升级风险预警

- 主动联系本身不是差评信号；是否及时解决问题更关键。
- 退款未成功、退款状态不清楚、较大金额退款诉求、物流明显延迟、买家多次追问或已提及 A-to-z，均按高风险订单跟进。
- 高风险订单必须指定明确下一步和检查时点，并保留统一处理时间线。
- 物流延迟可根据轨迹、延迟程度、库存和成本评估补偿、退款或补发，但不得与 Review、Feedback 或 A-to-z 撤销挂钩。
- 详细规则见 [[09-规则案例与实验/06-已验证结论/售后-退款争议与物流延迟客户风险识别]]。

## 罗马伞高频售后场景

- 只收到伞体或只收到底座。
- 第二箱仍在运输。
- 螺丝、摇柄、灯光组件等配件缺失。
- 大件运输破损。
- 买家未加重底座导致稳定性问题。
- 灯光系统或结构件故障。

## 合规底线

- 不诱导好评。
- 不要求删除或修改 Review。
- 不承诺未授权补偿。
- 不夸大产品防风、防水、认证或安全能力。
- 不得触发亚马逊平台违规风险；任何售后邮件都必须优先遵守 [[04-沟通合规规则]] 和 [[售后邮件撰写规则]]。

## 证据记录要求

每个售后案例至少记录：

| 字段 | 要求 |
|---|---|
| 订单号 | 不在公开文档中暴露完整买家隐私信息 |
| ASIN / 尺寸 / 是否带底座 | 必须和 [[02-产品事实库/00-产品矩阵总表]] 一致 |
| 问题类型 | 物流、破损、缺件、功能、风损、退款退货、评价申诉 |
| 证据 | tracking、送达截图、买家照片、仓库/物流回复 |
| 处理动作 | 解释、补发、部分退款、全额退款、退货标签、申诉 |
| 风险 | 合规风险、成本风险、复发风险、待确认信息 |

## 待确认统一归档

如果缺少产品事实、配件清单、功能细节、包装信息或合规判断，不得写成确认结论。

- 产品事实缺失：记录到 [[12-收集箱与附件/03-待确认产品信息]]
- 运营判断缺失：记录到 [[12-收集箱与附件/04-待验证观点]]
